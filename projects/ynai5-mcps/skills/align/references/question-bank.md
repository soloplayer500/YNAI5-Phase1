# ALIGN — Question Bank & Form Templates

Reference library of pre-built question sets by task type. Pull from these when building alignment forms. Always customize to the specific task — never use a template verbatim without adapting it to what Solo actually said.

---

## General Task Kickoff Questions

Use these when the task type is ambiguous or multi-domain.

**Scope:**
- "How big is this?" → [Quick output (1 thing) | Medium project (2–5 deliverables) | Full build (ongoing)]
- "What's the deadline or urgency?" → [Right now | Today | This week | No rush]
- "Have you tried this before?" → [First attempt | Tried and it didn't work | Partial progress already]

**Output format:**
- "What format do you need the output in?" → [Document / written | Code / technical | Strategy / plan | Visual / design | Conversational breakdown]
- "Who else will see this output?" → [Just me | My team | Public / audience | A client]

**Success bar:**
- "What does 'done well' look like for you?" → [Gets the job done, don't overthink it | Professional quality | Best possible version | Just a starting draft I'll refine]

## Content & Writing Tasks

*Triggered by: posts, articles, scripts, copy, emails, captions, threads, newsletters*

**Round 1 — Core alignment:**
- "What platform is this for?" → [TikTok | Instagram | X/Twitter | LinkedIn | YouTube | Telegram | Blog/newsletter | Other]
- "What's the goal of this content?" → [Build audience | Drive action/CTA | Educate | Entertain | Promote something | Establish credibility]
- "Tone?" → [Casual / street | Professional | Hype / energetic | Educational / breakdown | Storytelling | Straight to the point]

**Round 2 — Sharpen:**
- "Any existing content or style you want me to match?" → [Yes — I'll describe it | Yes — I'll paste an example | No — build from your read of my vibe | No — go with YNAI5 brand voice]
- "Length target?" → [Short / punchy (<150 words) | Medium (150–400 words) | Long-form (400+ words) | Whatever fits the platform]
- "Is there a specific hook or angle you already have in mind?" → [Yes — let me tell you | No — you pick the angle | Give me 3 options to choose from]

## Build / Development Tasks

*Triggered by: apps, bots, scripts, automations, dashboards, APIs*

**Round 1:**
- "What's the core function?" → [I'll describe it | It's like [X] but for [Y] | Automate something I do manually | Connect two tools together | Something new]
- "What stack or tools are involved?" → [Python | JavaScript / Node | Claude API | Telegram bot | Web app | I don't know — you recommend]
- "Does this connect to anything that already exists?" → [Yes — existing project (describe) | No — greenfield | Partially — extending something I have]

**Round 2:**
- "What's the MVP — what's the minimum it needs to do?" → [I'll define it | Just the core loop, nothing extra | Full version from the start | Prototype only]
- "Where does this live / run?" → [Local machine | Server / cloud | Telegram | Claude Code / Cowork | Browser / web | I don't know yet]

## Strategy & Planning Tasks

*Triggered by: roadmaps, business plans, launch plans, growth strategies*

**Round 1:**
- "What's the timeframe for this plan?" → [This week | This month | 90 days | 6 months | 1 year+]
- "What's the #1 outcome this plan needs to achieve?" → [Revenue / monetization | Audience growth | System efficiency | Brand positioning | Product launch]
- "What constraints should I know about?" → [Budget is tight | Time is tight | Solo operator — no team | Specific tools only | Other]

**Round 2:**
- "What's already been tried or decided?" → [Nothing yet — blank slate | Some things already locked in (I'll tell you) | Failed attempts I want to avoid repeating]
- "How detailed should the output be?" → [High-level overview | Step-by-step execution plan | Both: overview + detailed breakdown]

## Research & Analysis Tasks

*Triggered by: market research, competitive analysis, deep dives, summaries*

**Round 1:**
- "What decision does this research need to inform?" → [Buy / don't buy | Build / don't build | Which option to choose | Just want to understand something | Other]

- "Any sources or angles you specifically want included or avoided?" → [No preference | Include [X] | Avoid [Y] | Focus on [specific angle]]

## Reset Diagnostic Forms

*Use these only in MODE 2 — mid-conversation reset*

**Diagnostic Round 1:**
- "What went wrong?" → [Drifted from original goal | Wrong format or style | Wrong context given at start | Scope got too big or too small | Completely off base]
- "What do you want to preserve?" → [Nothing — full reset | Keep the research/context | Keep the structure | Keep most of it, fix specific part | Not sure]

**Diagnostic Round 2 (based on Round 1 answer):**

*If "drifted from original goal":*
- "What's the real goal we should be targeting?" → [Let me restate it | It's what I said originally — you just drifted | It evolved as we talked — here's where it landed]

*If "wrong format or style":*
- "What format do you actually need?" → [Show me an example | [Specific format] | You pick but explain your reasoning first]

*If "wrong context at start":*
- "What's the key context I was missing?" → [Let me tell you now | It's in a file I'll upload | It's something about the YNAI5 brand/ecosystem]

*If "scope too big/small":*
- "What's the right scope?" → [Smaller — just [specific piece] | Bigger — include [X] | Different — here's what it should actually cover]
