# Project Management Rules

When creating new project folders or updating existing ones:

- Every new project folder MUST have a `README.md` with: What It Is, Problem It Solves, Components table, Current Stage
- After creating a new project folder, update `projects/README.md` table with the new project, its status, and folder path
- After creating a new project folder, update the Workspace Map tree in `CLAUDE.md`
- Project status must use the standard emoji format: 🔨 Building, 🧪 Concept/Dev, 📋 Strategy, 🔄 Ongoing, ✅ Done, ⏸ Paused
- Projects with `session-log.md` should get a log entry whenever work is done on that project
- When a project status changes, update both the project's own README.md AND `projects/README.md`
